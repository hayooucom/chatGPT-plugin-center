Screening Smart Assistant AI Plug-in Market Access Guide

Web version: https://docs.qq.com/doc/p/0f631225a19893f306f849b45a0dae412de611b9

Application background description:

Nowadays, large-scale AI language model technology is widely used in fields such as text generation, but there are still many limitations in general task processing, such as inability to connect to the network, inability to interact with other services, and a single service method.

Therefore, we have done a lot of exploration on general tasks, including creating task services with one click.

Now we launch the AI plug-in market, which can be used with our general task processing web version (zyinfo.pro/agi).



Definition of AI plugin:

In the process of interaction between the user and the language model, related services need to be used, and the system automatically analyzes and calls the plug-in interface. Related services can also be invoked in general task processing.



The significance of our launch of the plug-in market:

For developers, they can explore interactive services with AI users without waiting for OpenAI's review. It is also possible to access more users. Participate in testing AI plug-ins earlier than peers to find the potential of related ecology.



New feature: One-sentence prompt can also be used as a plugin! Plug-in developers do not need to provide API services, but only provide a one-sentence prompt and plug-in description.



How to use the plugin:

There are currently two ways for users to access your plugin (you need to enable your plugin first):

(1) The user is in the AI dialogue scene and needs the service capability of the plug-in in the current context. Access and interact by connecting to the task interface that has been created.

(2) When creating a task, the user invokes the plug-in-related capabilities to create related task services, which can be used for a long time once created (except for temporary tasks created by creating task prompts, which need to be created with the taskset- command).



Plug-in docking instructions:

1. Plug-in information:

Plugin ID: Generated by plugin name and user id, unique

Website: The related website of this plug-in, so that users can understand the relevant information of the plug-in, more trust, and better use

Contact information: can communicate with you for feedback and communication

Description: The description of the plug-in, its functions and how to use it, how to wake it up in the AI service, etc. More descriptive content can be added. This content matches the latest instruction content when it participates in a user dialog or the context processing when a user task is generated. Developers can try multiple times to compare matching effects.

Task name: Create a task through the WeChat version or web version of Showing Smart Assistant, use the taskset command to create it, see https://docs.qq.com/doc/DS1VTblp4U1lyZWds?&u=403451cede784d1c82e357a25f1f8e28. Through the associated content of the plug-in, when the user-related needs are detected, the task is automatically executed. Tasks are a set of software services, generated by smart assistants. Once generated, the task can be called directly in the future, no need to generate again, and the execution speed is fast.

Processing instruction: This content will be added to the user's context (including when the user generates a task), and AI will process it. If there is a lot of content, it is recommended to use English. (You can add if the user's intention is..., then output in the format )

API document link: For the API document of the plug-in, please use https://platform.openai.com/tokenizer to test, and the content should not exceed 1500 tokens.

The content of the API document can be a yaml file (example) in openAPI format or a simple description, similar to this, if the content is too long, it is recommended to use English description.

API: Content Summary

Access point: http://127.0.0.1:8888

path: /summary?userid={userid}&content={content}&mission={mission}

content is the content you need to summarize, if the content length exceeds 10000, it needs to be truncated.

You can add mission descriptions, such as mission="summarize API-related information" and so on.

Return the summarized content (text format, not json)



API access point: start with http, access without authentication, we can provide userid (refer to above) information.

API information may not be provided. If API-related information is provided, the following items must be provided: API document link, access point, and name of the processing task (in English).

 展映智慧助手AI插件市场接入指南

网页版：https://docs.qq.com/doc/p/0f631225a19893f306f849b45a0dae412de611b9

应用背景说明：

如今大型AI语言模型技术在文本生成等领域有着广泛应用，但在通用任务处理上还是有诸多限制，比如无法连接网络、无法与其他服务交互、服务方式单一等。

因此我们在通用任务上做了很多探索，包括一键创建任务服务等。

现在我们推出AI插件市场，配合我们的通用任务处理网页版（zyinfo.pro/agi）使用。



AI插件的定义：

在用户与语言模型交互的过程中，需要用到相关服务，系统自动分析、调用插件接口。也可在通用任务处理中调用相关服务。



我们推出插件市场的意义：

对开发者来说，可以不用等待OpenAI的审核，探索与使用AI用户的互动服务。也可以接入更多用户。比同行更早一起参与测试AI插件，寻找相关生态的潜力。



新特性：一句话提示也能作为一个插件！ 插件开发者可以不用提供API服务，仅提供一句话提示和插件说明即可。



插件使用方式：

目前用户有两种方式接入你的插件（首先需要启用你的插件）：

	（1）用户在AI对话场景中，并且在当前语境中有需要与插件的服务能力。通过对接已经创建好的任务接口进行接入交互。

	（2）用户在 创建任务 中，调用插件相关的能力，创建相关任务服务，一经创建可以长期使用（通过创建任务提示语创建的临时任务除外，需要 taskset- 命令进行创建）。



插件对接说明：

1、插件信息：

插件ID： 由 插件名称和用户id 生成，唯一

网站：本插件的相关网站，让用户了解插件相关信息，更多信任，更好使用

联系方式：可以与你们进行反馈和交流

描述：插件的说明，功能和简单的使用方式，如何在AI服务中唤醒等。可以添加更多的说明内容。此内容参与到用户对话时匹配最新的指令内容或用户任务生成时的上下文处理。开发者可以多次尝试对比匹配效果。

任务名称：通过 展映智慧助手微信版 或网页版 创建的任务，使用taskset命令创建 ，参看https://docs.qq.com/doc/DS1VTblp4U1lyZWds?&u=403451cede784d1c82e357a25f1f8e28。通过插件的关联内容，检测到用户相关需求时，自动执行该任务。任务是一套软件服务，由智慧助手生成。生成一次，以后直接调用该任务即可，无需再生成，执行速度快。

处理指令：此内容会添加到用户的上下文中（包括用户生成任务时），AI进行相关处理。若内容较多，推荐使用英文。（可以添加 若用户的意图是。。。，那么按格式输出 ）

API文档链接：插件的API文档，请使用https://platform.openai.com/tokenizer 测试，内容不要超过 1500token。

API文档内容可以是openAPI格式的yaml文件（示例）也可以简单描述，类似这样，如果内容过长，建议使用英文描述。

API：内容摘要

path： /summary?userid={userid}&content={content}&mission={mission}

content是你需要总结的内容，如果内容长度超过10000需要截断。

可以添加任务说明，如mission=“总结API相关的信息”等。

返回概括的内容（文本格式，非json）
